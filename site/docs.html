Security model

  • Zero-knowledge: vault contents are encrypted client-side.
  • Key derivation: Argon2id (memory-hard) via argon2-cffi.
  • Encryption: AES-256-GCM (authenticated encryption) via cryptography.
  • HIBP breach checks: only SHA-1 prefixes are transmitted (k-anonymity).

Planned vault format

Single encrypted file for portability:
vault.enc (Encrypted blob) - salt - Argon2id params - AES-GCM nonce - ciphertext - authentication tag
