# Security Policy

This repository focuses on **security review and privacy guidance** for Instagram accounts.

## Supported reporting

Please report issues responsibly.

### What to include
- Impact (what could go wrong)
- Steps to reproduce (only if safe)
- Screenshots/logs (redact personal data)
- Suggested mitigation

## What not to do

- Do not publish exploit code or instructions publicly.
- Do not attempt unauthorized access to any third-party systems.

## Security assurance statement

This project is documentation-first. If tooling is added later, it must follow these constraints:
- No credential capture (passwords, cookies, tokens)
- No scraping or unauthorized collection
- Least-privilege design
- Clear data handling and retention guidance
